1、提示音音源说明：
使用百度语音tts合成：
链接：http://ai.baidu.com/tech/speech/tts
下载的是16K采样率，需要转采样为44.1k文件。

2、使用说明：
提示音需要使用下文工具合成all.bin 并输出sound_remind_item.h 预定义文件。音频SDK工程已自动转换导入工程源文件（script.bat），用户工程需自行迁移。

3、工具：
\ALI 目录下存放提示音源文件，文件名可以字母、数字、‘_’，后缀mp3/wav/sbc。特别提醒，不可用中文。
\temp 是临时文件夹，随时可清空。

运行MergeAudio2BinNew.exe后生成 all.bin 和sound_remind_item.h文件（从文件名中截取前八字符作为flash中条目查询字符串，不足时补空格，缺省地址配置是0x100000）
前者使用o26_download工具下载到flash，后者拷贝到代码工程，SDK中由（script.bat）自动转换文件名后实施。
点击(Flash Burner) AndeSight toolbar，在对话框Flash Driver，设置o26_download（含路径）
设置Flash Image为all.bin（含路径）。
设置Programming Start Address为constdata地址如：0x120000.
点击burn。

Flash const data数据burn完毕 及时把burn地址改为0，避免意外。


要点：
1、提示音Demo 实现了SBC和MP3两种提示音使用方式。
2、AudioPlay实现提示音播放、插播、复位功能。要求使用解码器的APP模式实现播控分离后才调用提示音功能。
3、AudioPlay提示音当前的MP3方案最节约ram和flash资源，播控分离带来一定的逻辑复杂性，插播提示音代码一定是running阶段的app消息处理体内，其他位置只能发消息通知。
4、避免buf溢出，提示音文件格式不要混用，程序中预选了文件格式。
5、代码工程里做了文件名转换，统一命名方式。
6、需要提示音bin存储位置调整的，使用命令行 MergeAudio2BinNew.exe --addr 0x???,代码中协同。音频SDK的script.bat是自动脚本文件，调整提示音存储区参数时，请修改脚本。
7、需要调整提示音导入文件路径，使用命令行参数 MergeAudio2BinNew.exe --inputdir ..\..\ALI_other，其中每一个"..\"代表一层上级目录；也可以直接填写绝对路径，比如--inputdir C:\mp3
8、支持win7，win10操作系统
